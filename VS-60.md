# Plan: Bidirectional LSP Navigation for Soul Markdown Docs

## Context

The `soul-lsp` server currently provides hover, goto-definition, and find-references only when the cursor is on a code annotation. All three handlers call `annotation_at()` and return `None` immediately if the cursor is in any other file — including Soul markdown docs. This means opening a `.docs/**/*.md` Soul document gives no LSP intelligence at all: you can't jump to the annotated code, see what implements it, or hover for a summary.

The fix adds a `document_at()` fallback so all three handlers work bidirectionally — from doc → code as well as code → doc.

---

## Critical Files

- `crates/soul-lsp/src/server.rs` — required; all LSP changes live here
- `crates/indexer/src/mcp/mod.rs` — optional (Step 5 only); update `soul_explain` description strings

## Implementation

### Step 1 — Add `document_at` helper

Insert after the closing `}` of `linked_doc` (line 57), before `to_uri` (line 59):

```rust
fn document_at<'g>(graph: &'g SemanticGraph, root: &Path, uri: &Uri) -> Option<&'g Document> {
    let req_path = uri.to_file_path()?;
    let req_path = req_path.canonicalize().unwrap_or(req_path.to_path_buf());
    graph.documents.iter().find(|d| {
        let abs = if d.path.is_absolute() {
            d.path.clone()
        } else {
            root.join(&d.path)
        };
        let canon = abs.canonicalize().unwrap_or(abs);
        canon == req_path
    })
}
```

Same shape as `annotation_at` — resolves path, ignores line/character (any position in the doc triggers it).

### Step 2 — Update `hover`

Replace the early-return `else` branch with a doc fallback:

```rust
// BEFORE
let Some(ann) = annotation_at(...) else {
    return Ok(None);
};
let mut md = format!("**{}**", ann.id);
if let Some(doc) = linked_doc(graph, &ann.id) {
    if let Some(title) = &doc.title {
        md.push_str(&format!("\n\n*{title}*"));
    }
    md.push_str(&format!("\n\n`{}`", doc.path.display()));
}

// AFTER
let md = if let Some(ann) = annotation_at(graph, &self.root, &pos.text_document.uri, pos.position.line) {
    let mut md = format!("**{}**", ann.id);
    if let Some(doc) = linked_doc(graph, &ann.id) {
        if let Some(title) = &doc.title {
            md.push_str(&format!("\n\n*{title}*"));
        }
        md.push_str(&format!("\n\n`{}`", doc.path.display()));
    }
    md
} else if let Some(doc) = document_at(graph, &self.root, &pos.text_document.uri) {
    let mut md = format!("**{}**", doc.id);
    if let Some(title) = &doc.title {
        md.push_str(&format!("\n\n*{title}*"));
    }
    let locs: Vec<_> = graph.annotations.iter().filter(|a| a.id == doc.id).collect();
    if !locs.is_empty() {
        md.push_str(&format!("\n\n{} code location(s):", locs.len()));
        for a in &locs {
            md.push_str(&format!("\n- `{}:{}`", a.path.display(), a.line));
        }
    }
    md
} else {
    return Ok(None);
};
```

Result: hovering anywhere in a Soul doc shows id, title, and a list of annotated files+lines.

### Step 3 — `goto_definition` — add fallback to annotations

From a code annotation `goto_definition` already jumps to the Soul doc. The reverse direction — from a Soul doc, jumping to all annotated code locations — is not currently supported, but should be for full bidirectionality.

Add a doc fallback that returns all annotation locations (using `GotoDefinitionResponse::Array`):

```rust
// BEFORE
let Some(ann) = annotation_at(graph, &self.root, &pos.text_document.uri, pos.position.line)
else {
    return Ok(None);
};
let Some(doc) = linked_doc(graph, &ann.id) else {
    return Ok(None);
};
let Some(uri) = to_uri(&self.root, &doc.path) else {
    return Ok(None);
};
Ok(Some(GotoDefinitionResponse::Scalar(Location {
    uri,
    range: Range::default(),
})))

// AFTER
let locations_from_id = |id: &str| -> Option<GotoDefinitionResponse> {
    let locs: Vec<Location> = graph
        .annotations
        .iter()
        .filter(|a| a.id == id)
        .filter_map(|a| {
            let uri = to_uri(&self.root, &a.path)?;
            let line = (a.line as u32).saturating_sub(1);
            Some(Location {
                uri,
                range: Range {
                    start: Position { line, character: 0 },
                    end: Position { line, character: 0 },
                },
            })
        })
        .collect();
    if locs.is_empty() { None } else { Some(GotoDefinitionResponse::Array(locs)) }
};

if let Some(ann) = annotation_at(graph, &self.root, &pos.text_document.uri, pos.position.line) {
    let Some(doc) = linked_doc(graph, &ann.id) else {
        return Ok(None);
    };
    let Some(uri) = to_uri(&self.root, &doc.path) else {
        return Ok(None);
    };
    Ok(Some(GotoDefinitionResponse::Scalar(Location {
        uri,
        range: Range::default(),
    })))
} else if let Some(doc) = document_at(graph, &self.root, &pos.text_document.uri) {
    Ok(locations_from_id(&doc.id))
} else {
    Ok(None)
}
```

Result: goto-definition from a Soul doc opens a picker showing all annotated code locations, symmetric with find-references.

### Step 4 — Update `references`

Same pattern — add doc fallback after the `annotation_at` branch:

```rust
// BEFORE
let Some(ann) = annotation_at(...) else {
    return Ok(None);
};
let id = ann.id.clone();
// → filters annotations by id

// AFTER
let id = if let Some(ann) = annotation_at(graph, &self.root, &pos.text_document.uri, pos.position.line) {
    ann.id.clone()
} else if let Some(doc) = document_at(graph, &self.root, &pos.text_document.uri) {
    doc.id.clone()
} else {
    return Ok(None);
};
// existing filter+collect unchanged below
```

Result: find-references from a Soul doc lists all code locations — same set of annotation locations as goto-definition (Step 3), surfaced via the references UI instead.

## Semantic Classification

Soul's model has a clean definition/reference split:

| Concept | Role |
|---|---|
| Soul document (`.md`) | **Definition** — the authoritative specification. Exactly one per ID. |
| Code annotation (`#[soul(...)]`, `[Soul(...)]`) | **Reference** — a claim that a piece of code implements or participates in that concept. Many per ID, across any language. |

The existing LSP code handles annotation → doc (goto_definition) and annotation → all annotations (references). This plan makes the reverse directions symmetric: doc → all annotation locations (goto_definition) and doc → all annotation locations (references), plus doc → hover summary.
