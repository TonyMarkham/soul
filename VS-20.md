# Plan: Config + Index Artifact

## Context

The indexer currently hardcodes exclusion lists and annotation extensions directly in `scan/mod.rs`. Every `explain` call does a full live scan, making future MCP/LSP consumers impractical. This slice adds:

1. **`SoulConfig`** — loaded from `.soul/soul.toml`, replaces all hardcoded scan rules
2. **Index artifact** — `indexer index` writes `.soul/index.db` (SQLite via sqlx); `indexer explain` queries it directly (falls back to live scan if absent)

Soul scans all markdown files in the repository subject to exclusion rules — there is no `docs_dir` concept. A markdown file is a Soul document if it has valid frontmatter; otherwise it is ignored.

## Step 4 — New error variants

In `crates/indexer/src/error/mod.rs`, add variants and constructors following the **exact existing pattern**: `#[track_caller]` + `ErrorLocation::from(Location::caller())`.

New variants:
```rust
#[error("{location} failed to read config `{path}`: {source}")]
ConfigRead {
    path: PathBuf,
    location: ErrorLocation,
    #[source]
    source: std::io::Error,
},

#[error("{location} failed to parse config `{path}`: {source}")]
ConfigParse {
    path: PathBuf,
    location: ErrorLocation,
    source: toml::de::Error,
},

#[error("{location} index db error at `{path}`: {source}")]
IndexDb {
    path: PathBuf,
    location: ErrorLocation,
    #[source]
    source: sqlx::Error,
},
```

New constructors:
```rust
#[track_caller]
pub fn config_read(path: PathBuf, source: std::io::Error) -> Self {
    Self::ConfigRead { path, location: ErrorLocation::from(Location::caller()), source }
}

#[track_caller]
pub fn config_parse(path: PathBuf, source: toml::de::Error) -> Self {
    Self::ConfigParse { path, location: ErrorLocation::from(Location::caller()), source }
}

#[track_caller]
pub fn index_db(path: PathBuf, source: sqlx::Error) -> Self {
    Self::IndexDb { path, location: ErrorLocation::from(Location::caller()), source }
}
```

Note: `toml::de::Error` implements `std::error::Error` in the version used (toml ≥ 0.9 / 1.x). Include `#[source]` on the field exactly as shown above.

## Step 5 — `Display` / `FromStr` on serialized enums

`AnnotationSyntax` and `DiagnosticSeverity` are stored as `TEXT` in SQLite. Their string representations must come from `Display` impls — no raw string literals in query code.

Add to `crates/indexer/src/model/annotation_syntax.rs`:
```rust
impl std::fmt::Display for AnnotationSyntax {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        match self {
            Self::RustAttribute => f.write_str("rust_attribute"),
            Self::CSharpAttribute => f.write_str("csharp_attribute"),
        }
    }
}

impl std::str::FromStr for AnnotationSyntax {
    type Err = String;

    fn from_str(s: &str) -> Result<Self, Self::Err> {
        match s {
            "rust_attribute" => Ok(Self::RustAttribute),
            "csharp_attribute" => Ok(Self::CSharpAttribute),
            other => Err(format!("unknown annotation syntax: `{other}`")),
        }
    }
}
```

Same pattern for `DiagnosticSeverity` in `model/diagnostic_severity.rs` (`"warning"` | `"error"`).

Query code uses `.to_string()` when binding and `.parse()` when reading rows — no enum string literals in query code.

## Step 7 — Index artifact (SQLite)

New files:
```
crates/indexer/migrations/
  0001_initial.up.sql
crates/indexer/src/index/
  mod.rs
```

### Migration: `0001_initial.up.sql`

```sql
CREATE TABLE documents (
    id      TEXT NOT NULL,
    kind    TEXT NOT NULL,
    title   TEXT,
    path    TEXT NOT NULL
);

CREATE TABLE annotations (
    id       TEXT NOT NULL,
    role     TEXT,
    metadata TEXT NOT NULL,
    path     TEXT NOT NULL,
    line     INTEGER NOT NULL,
    syntax   TEXT NOT NULL,
    raw      TEXT NOT NULL
);

CREATE TABLE diagnostics (
    severity TEXT NOT NULL,
    path     TEXT NOT NULL,
    line     INTEGER,
    message  TEXT NOT NULL
);
```

sqlx creates and maintains `_sqlx_migrations` automatically. Pending migrations run on every connect. Future schema changes get a new numbered file — existing ones are never edited.

```rust
sqlx::migrate!("./migrations").run(&pool).await
    .map_err(|e| IndexerError::index_db(index_path.clone(), e.into()))?;
```

(`sqlx::migrate!` path is relative to the crate root, resolved at compile time.)

### Public API (async)

All return `IndexerResult<T>`. Use `sqlx::query()` — no `DATABASE_URL`, no compile-time macros.

```rust
pub async fn write_index(root: &Path, graph: &SemanticGraph) -> IndexerResult<PathBuf>
pub async fn open_index(root: &Path) -> IndexerResult<Option<sqlx::SqlitePool>>
pub async fn load_graph(pool: &sqlx::SqlitePool) -> IndexerResult<SemanticGraph>
pub async fn explain_from_index(pool: &sqlx::SqlitePool, id: &str) -> IndexerResult<ExplainResult>
```

- `write_index`: `create_dir_all(root.join(constants::SOUL_DIR))` → connect → run migrations → `DELETE FROM` all three tables → insert all rows in a single transaction → return index path
- `open_index`: return `None` if index file does not exist; connect, run migrations, return `Some(pool)`
- `load_graph`: `SELECT` all rows from all three tables; reconstruct `SemanticGraph` via direct struct initialization; enums via `.parse()`
- `explain_from_index`: `SELECT ... WHERE id = ?` on documents and annotations; all diagnostics (no filter)

Internal helpers are private `fn`; `pub(crate)` only if needed across submodules.

### SQLite connection URL

`SqlitePool::connect` requires a URL string, not a `PathBuf`. Construct it in both `write_index` and `open_index`:

```rust
let url = format!(
    "sqlite://{}",
    root.join(constants::SOUL_DIR).join(constants::INDEX_FILE).display()
);
let pool = SqlitePool::connect(&url).await
    .map_err(|e| IndexerError::index_db(index_path.clone(), e))?;
```

On Windows the path separator will be wrong — use `.to_string_lossy()` and note this is a known limitation for now.

use indexer::{IndexerError, IndexerResult, constants};

pub async fn run(root: &Path) -> IndexerResult<()> {
    let soul_dir = root.join(constants::SOUL_DIR);
    let config_path = soul_dir.join(constants::CONFIG_FILE);

    if config_path.exists() {
        return Err(IndexerError::cli(format!(
            "already initialised: `{}` exists",
            config_path.display()
        )));
    }

    std::fs::create_dir_all(&soul_dir)
        .map_err(|e| IndexerError::config_read(config_path.clone(), e))?;

    std::fs::write(&config_path, indexer::constants::EXAMPLE_CONFIG)
        .map_err(|e| IndexerError::config_read(config_path.clone(), e))?;

    println!("Initialized → {}", config_path.display());
    Ok(())
}

**`commands/explain.rs`:**
```rust
use std::path::Path;

use indexer::{ExplainResult, IndexerResult, explain, load_config, open_index,
              explain_from_index, scan_repository};

pub async fn run(root: &Path, id: &str) -> IndexerResult<()> {
    let result = match open_index(root).await? {
        Some(pool) => explain_from_index(&pool, id).await?,
        None => {
            let config = load_config(root)?;
            let graph = scan_repository(root, &config)?;
            explain(&graph, id)
        }
    };
    print_result(&result);
    Ok(())
}

fn print_result(result: &ExplainResult) {
    println!("ID: {}", result.id);
    println!();
    println!("Documents:");
    if result.documents.is_empty() {
        println!("none");
    } else {
        for doc in &result.documents {
            match &doc.title {
                Some(title) => println!("- {} [kind={}, title={}]", doc.path.display(), doc.kind, title),
                None => println!("- {} [kind={}]", doc.path.display(), doc.kind),
            }
        }
    }
    println!();
    println!("Annotations:");
    if result.annotations.is_empty() {
        println!("none");
    } else {
        for ann in &result.annotations {
            match &ann.role {
                Some(role) => println!("- {}:{} [role={}]", ann.path.display(), ann.line, role),
                None => println!("- {}:{}", ann.path.display(), ann.line),
            }
        }
    }
    if !result.scan_diagnostics.is_empty() {
        println!();
        println!("Diagnostics:");
        for diag in &result.scan_diagnostics {
            match diag.line {
                Some(line) => println!("- {}:{} {}", diag.path.display(), line, diag.message),
                None => println!("- {} {}", diag.path.display(), diag.message),
            }
        }
    }
}
```
