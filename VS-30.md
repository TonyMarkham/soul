# VS: Soul MCP Server

## Context

The Soul indexer core is complete â€” it scans repos, parses annotations and frontmatter, builds a
semantic graph, and stores it in SQLite. The missing piece is exposing that graph to an LLM. An MCP
server is the bridge: once deployed, Claude Code can query Soul to discover what's documented, what
isn't, and what code has no linked docs â€” enabling it to write documentation and add annotations
autonomously.

The MCP server is a new `serve` command on the existing `indexer` binary â€” no new crate, no new
executable. The binary already lives in `.soul/indexer` by convention, so `.mcp.json` points there.
Transport is stdio (JSON-RPC 2.0). Claude Code spawns the process and keeps it alive for the session.

## `SoulServer` (`src/mcp/mod.rs`)

```rust
#[derive(Clone)]
struct SoulServer { root: PathBuf }
```

Implement `ServerHandler` (name: `"soul"`, version from `env!("CARGO_PKG_VERSION")`).

Use `#[tool_router]` macro. Four tools:

### `soul_index`
1. `load_config(&root)` â†’ config
2. `scan_repository(&root, &config)` â†’ graph
3. `write_index(&root, &graph)`
4. Return: `"Indexed {n} documents, {n} annotations, {n} diagnostics."`

### `soul_explain { id: String }`
1. `open_index(&root)` â†’ if Some(pool): `explain_from_index(&pool, &id)`
2. Else: `load_config` + `scan_repository` + `explain(&graph, &id)`
3. Return: formatted via `format::explain_result`

### `soul_list_documents`
1. `open_index(&root)` â†’ if Some(pool): `load_graph(&pool)` else scan
2. Return: one line per document â€” `{id} [{kind}] {title} â€” {path}`

### `soul_list_gaps`
1. Load graph (index-or-scan)
2. `unlinked`: annotation IDs with no matching document
3. `undocumented`: document IDs with no matching annotation
4. Return: two labelled sections

## `format.rs` (`src/mcp/format.rs`)

Plain-text formatters (LLM-readable):
- `explain_result(result: &ExplainResult) -> String`
- `document(doc: &Document) -> String`
- `annotation(ann: &CodeAnnotation) -> String`
